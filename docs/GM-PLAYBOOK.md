# GM 운영 가이드

## 1. 공통 운영 흐름
1. 카탈로그에서 게임 선택
2. 상세 프리뷰 확인
3. `입장`으로 GM 화면 진입
4. 슬라이드 이동
5. 필요 시 규칙 팝업/헤더 액션/타이머 사용

## 2. GM 화면 조작 포인트
- 하단 floating bar의 `이전 / 다음` 버튼: 슬라이드 이동
- `홈`: 카탈로그로 이동
- 화면 상단의 제목: 슬라이드 `data-title` 기준으로 표시되며, 본문 제목과 중복되면 본문 쪽 제목은 자동으로 숨김
- 키보드 좌우키: 슬라이드 이동
- 스와이프: 터치 환경에서 슬라이드 이동
- `현재 룰`: 현재 슬라이드와 연결된 단일 규칙 열기
- `전체 룰`: 게임 전체 규칙 탭 팝업 열기
- 게임별 액션 버튼: 게임별 특화 동작 실행
- `더보기`: BGM이 활성화된 게임에서 현재 BGM 이름, 재생/일시정지, 음량 조절 열기
- 진행 확인 다이얼로그: 슬라이드에 `data-next-confirm-*`가 있으면 다음 이동 전에 확인 모달 표시

## 3. 타이머 운영
- 기본 타이머와 플레이어별 타이머를 지원합니다.
- 슬라이드 이동 시 현재 슬라이드의 타이머만 조작합니다.
- 탭 비활성화나 브라우저 숨김 상태에서는 타이머가 자동 일시정지됩니다.
- 플레이어별 타이머는 현재 플레이어 종료 후 다음 플레이어로 자동 이동합니다.

## 4. 규칙 팝업 운영
- 규칙 데이터는 게임별로 다를 수 있습니다.
- `전체 룰` 팝업은 `game.json > rules.tabs` 구성을 따릅니다.
- `현재 룰` 버튼은 현재 슬라이드의 `data-part-info-topic` 또는 `data-part-info-title`과 연결됩니다.
- `전체 룰`은 현재 슬라이드 전용 규칙이 아니라 게임 전체 규칙을 여는 버튼으로 유지합니다.
- 권장 방식은 `data-part-info-topic`을 사용해, 현재 단계에서 바로 필요한 규칙 topic 하나를 명시적으로 연결하는 것입니다.
- 예:
  - 설정서 숙지 시간 -> `설정서와 비공개 정보`
  - 제1조사 / 제2조사 -> `조사 페이즈`
  - 투표 -> `투표 페이즈`

## 5. 헤더 액션 운영
- 게임별 추가 버튼은 게임 패키지의 `headerActions[]`에서 하단 floating bar에 동적으로 렌더링됩니다.
- 공통 HTML에 게임 특화 버튼을 직접 추가하지 않습니다.
- 현재는 `confirmJump` 액션을 지원합니다.
- 액션 종류가 늘어나면 선언 구조는 유지하고 `action.type`만 확장합니다.

## 6. 문의 기능
- 하단 `contact us` 버튼에서 문의 모달을 열 수 있습니다.
- 문의 유형:
  - `권리자 삭제 요청`
  - `피드백`
- 현재는 `mailto:` 기반으로 메일 앱을 엽니다.
- 기본 메일 앱이 없는 환경에서는 열리지 않을 수 있습니다.

## 7. 게임별 차이 처리 원칙
- 게임마다 파트 수, 순서, 타이머 시간, 규칙 탭 구성이 달라도 공통 렌더러를 우선 사용합니다.
- 예외 기능은 `game.json` 데이터나 외부 `rules.json`으로 분리합니다.
- `app.js`에 특정 게임 이름으로 하드코딩 분기를 추가하는 방식은 지양합니다.
- `"어떤 지시 전에는 다음으로 넘어가지 마시오"` 유형은 슬라이드의 `data-next-confirm-title`, `data-next-confirm-detail`, `data-next-confirm-note`로 처리합니다.
